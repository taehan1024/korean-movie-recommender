# Korean Movie Recommender

[![Streamlit App](https://static.streamlit.io/badges/streamlit_badge_black_white.svg)](https://korean-movie-recommender-dbs65tfqxee5tdixtse4o6.streamlit.app)

Cross-cultural content-based movie recommender: enter a US movie, get ranked Korean movie recommendations.

Built with Python, TMDB API, sentence-transformers, and Streamlit. No user interaction data — purely content-based using plot synopses, genres, keywords, and year proximity.

**[Try the live demo](https://korean-movie-recommender-dbs65tfqxee5tdixtse4o6.streamlit.app)**

## Key Results

| Metric | V1 Best | V2 Hybrid (tuned) | Change |
|---|---|---|---|
| DCG@10 | 0.173 | **0.298** | **+72%** |
| MRR | -- | **0.171** | new metric |
| NDCG@10 | 0.066 | **0.123** | +86% |
| Thematic Hit@10 | -- | **0.250** | 3x vs pre-keywords |
| Remake Hit@10 | -- | **0.667** | strong |

**Key insight:** TMDB keywords (e.g., "serial killer", "time travel", "revenge") tripled thematic match detection. Cast/crew features were confirmed as noise (0.8% cross-industry overlap, optimal weight = 0.0). Grid search over 288 weight combinations found: `text=0.47, genre=0.24, keyword=0.18, cast=0.00, year=0.11`.

## Quick Start

```bash
# 1. Clone and setup
git clone https://github.com/taehan1024/korean-movie-recommender.git
cd korean-movie-recommender
make setup

# 2. Add your TMDB API key
echo "TMDB_API_KEY=your_key_here" > .env

# 3. Run the pipeline
make ingest           # Fetch 5000 US + 1512 KR movies from TMDB
make fetch-keywords   # Add TMDB keywords to movie catalogs
make features-v2      # Build all feature matrices
make eval-v2          # Benchmark models on gold evaluation set

# 4. Launch the app
make app-v2           # Opens Streamlit UI at localhost:8501
```

Get a free TMDB API key at [themoviedb.org/settings/api](https://www.themoviedb.org/settings/api).

### V1 vs V2 Files

This project was developed iteratively. V1 files (`models.py`, `evaluate.py`, etc.) contain the initial implementation. V2 files contain improvements informed by literature review and evaluation analysis:

| Change | Why |
|---|---|
| Added keyword Jaccard similarity | Genres are coarse (19 categories); keywords capture thematic nuance |
| Added year proximity (Gaussian decay) | Era similarity matters for cross-cultural matching |
| Grid-searched hybrid weights | Data-driven weight allocation instead of heuristic |
| Dropped cast features (weight=0.0) | Only 0.8% person overlap between US and KR catalogs |
| Added Hit@K, MRR metrics | P@K is misleading with ~3 gold matches per query |
| Per-relevance evaluation | Separate metrics for remakes, thematic matches, and genre matches |

## Models

| Model | Description | DCG@10 |
|---|---|---|
| TF-IDF + Cosine | Bag-of-words baseline on plot synopses | 0.134 |
| Sentence Embeddings | Semantic similarity via all-MiniLM-L6-v2 (384-dim) | 0.173 |
| **Hybrid v2** | **Weighted fusion of text + genre + keyword + year** | **0.298** |

The hybrid model applies min-max normalization to each feature dimension, then combines with grid-search-tuned weights. See [report.md](report.md) for details.

## Evaluation

**Gold set:** 174 curated US-KR movie pairs across 3 relevance levels:
- **Remake (3):** Direct remakes/adaptations (e.g., Oldboy US 2013 ↔ Oldboy KR 2003)
- **Thematic (2):** Similar themes/tone (e.g., Zodiac ↔ Memories of Murder)
- **Genre (1):** Same genre, some topical overlap (e.g., Gravity ↔ Space Sweepers)

**Primary metric:** DCG@10 (unnormalized), following [Jeunen et al. (KDD 2024)](https://arxiv.org/abs/2307.06517) — nDCG normalization can invert method ordering.

**Methodology grounded in:**
- [DaisyRec 2.0](https://arxiv.org/abs/2206.10848) — evaluation rigor, appropriate metrics for sparse labels
- [ReasoningRec](https://arxiv.org/abs/2312.04482) — item descriptions dominate in sparse/no-user-data settings
- [Jeunen et al.](https://arxiv.org/abs/2307.06517) — DCG over nDCG for model selection

## Design Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Data source | TMDB API | Plots, genres, keywords, cast, crew, posters in one API |
| Text model | all-MiniLM-L6-v2 | Free, 80MB, good paraphrase handling |
| Similarity | numpy cosine | 1512 KR movies = milliseconds; FAISS overkill |
| Primary metric | DCG@10 | nDCG normalization can invert rankings (Jeunen et al.) |
| No multilingual model | English-only | TMDB provides English overviews for 99.9% of KR catalog |
| No collaborative filtering | Content-based only | Zero user interaction data to exploit |
